# Cleanup - Worktree Close Workflow

Remove a worktree and optionally delete the branch.

## Prerequisites

- PR has been merged (or you want to abandon the worktree)
- No meaningful uncommitted changes in the target worktree

## Permission & Safety Philosophy

**Goal: minimize permission prompts while staying safe.**

- When the PR is confirmed **MERGED** (via cache or API), all cleanup operations are safe —
  proceed without asking. The merge is an irreversible GitHub state.
- Only ask for confirmation when the PR is NOT merged (OPEN, CLOSED, NONE).
- **One exception**: if the issue/PR lists manual post-merge validations (step 2b-ii), pause
  and check with the user before removing the worktree — that's a "did you test it?" question,
  not a permission prompt.
- Combine as many operations as possible into single bash commands to reduce prompts.

### Handling uncommitted changes

Not all uncommitted changes are equal. Auto-restore **generated/tooling files** that aren't
real work, but **stop and ask** if there are changes that look like they could be meaningful.

Auto-restore (safe to discard — regenerated by tooling):
- `.envrc`, `.direnv/` — direnv-generated
- `.claude/` — Claude session state files within the worktree

If the **only** uncommitted changes are in the auto-restore list, run
`git -C "$CURRENT_WORKTREE" checkout -- <files>` silently and continue.

If there are **any other** uncommitted changes (source code, config, etc.), stop and ask
the user — these may be unpushed work that was left behind.

## Critical: Never cd into the target worktree

**IMPORTANT**: The Bash tool validates the current working directory before running any command.
If the worktree directory is deleted, ALL subsequent commands fail with "Path does not exist".

**Solution**: Resolve the target worktree path and branch using `git worktree list` and `git -C`,
but always stay in (or switch to) main. Never cd into the target worktree.

## Workflow

### 1. Resolve Target and Capture Context (SINGLE COMMAND)

Supports three modes:
- **Argument provided, running from main**: resolve glob relative to CWD (e.g. `../4*` or `./4*`)
- **Argument provided, running from another worktree**: resolve glob relative to CWD
- **No argument, running from inside a worktree**: use current worktree as target

Uses `git worktree list` to look up the branch for the resolved path. Never cd's into the target.

```bash
MAIN_WORKTREE=$(git worktree list --porcelain | grep '^worktree ' | head -1 | cut -d' ' -f2)

ARG="$ARGUMENTS"
if [ -n "$ARG" ]; then
  # --- Argument provided: resolve glob to find target ---
  PATTERN="$ARG"
  case "$PATTERN" in
    *\*) ;; # already has wildcard
    */) PATTERN="${PATTERN}*" ;;
    *) PATTERN="${PATTERN}*" ;;
  esac

  # Use eval so globs in variables expand in zsh; newline list avoids array indexing differences
  MATCH_LIST=$(eval ls -d "${PATTERN}/" 2>/dev/null)
  MATCH_COUNT=$(echo "$MATCH_LIST" | grep -c .)

  if [ "$MATCH_COUNT" -eq 0 ] || [ -z "$MATCH_LIST" ]; then
    echo "ERROR: No directory matching '${PATTERN}' found"
    exit 1
  elif [ "$MATCH_COUNT" -gt 1 ]; then
    echo "ERROR: Ambiguous match for '${PATTERN}'. Multiple directories found:"
    echo "$MATCH_LIST" | sed 's/^/  /'
    exit 1
  fi

  # Resolve to absolute path (MATCH_LIST has exactly one entry)
  CURRENT_WORKTREE=$(cd "$MATCH_LIST" && pwd)
  echo "Resolved '$ARG' → $CURRENT_WORKTREE"
else
  # --- No argument: must be running from inside a worktree ---
  COMMON_DIR=$(git rev-parse --git-common-dir 2>/dev/null)
  GIT_DIR=$(git rev-parse --git-dir 2>/dev/null)
  if [ "$COMMON_DIR" = "$GIT_DIR" ]; then
    echo "ERROR: No argument provided and not in a worktree."
    echo "Usage: /cleanup <glob>  (e.g. /cleanup ../4*)"
    exit 1
  fi
  CURRENT_WORKTREE=$(pwd)
fi

# Validate target is not main
if [ "$CURRENT_WORKTREE" = "$MAIN_WORKTREE" ]; then
  echo "ERROR: Target resolves to main worktree. Cannot clean up main."
  exit 1
fi

# Look up branch from git worktree list (works without cd'ing into target)
CURRENT_BRANCH=$(git worktree list --porcelain | awk -v wt="$CURRENT_WORKTREE" '
  $1 == "worktree" && $2 == wt { found=1 }
  found && $1 == "branch" { sub(/^refs\/heads\//, "", $2); print $2; exit }
')

if [ -z "$CURRENT_BRANCH" ]; then
  echo "ERROR: '$CURRENT_WORKTREE' is not a registered git worktree"
  exit 1
fi

# Check for uncommitted changes using git -C (no cd needed)
DIRTY_FILES=$(git -C "$CURRENT_WORKTREE" status --porcelain 2>/dev/null)
if [ -n "$DIRTY_FILES" ]; then
  # Separate auto-restorable files from meaningful changes
  AUTO_RESTORE_PATTERN='^\s*\S+\s+(\.envrc|\.direnv/|\.claude/)'
  MEANINGFUL=$(echo "$DIRTY_FILES" | grep -vE "$AUTO_RESTORE_PATTERN" || true)
  AUTO_ONLY=$(echo "$DIRTY_FILES" | grep -E "$AUTO_RESTORE_PATTERN" || true)

  if [ -n "$AUTO_ONLY" ]; then
    echo "Auto-restoring generated files:"
    echo "$AUTO_ONLY"
    # Restore each auto-restorable file
    echo "$AUTO_ONLY" | awk '{print $2}' | while read -r f; do
      git -C "$CURRENT_WORKTREE" checkout -- "$f" 2>/dev/null || true
    done
  fi

  if [ -n "$MEANINGFUL" ]; then
    echo "WARNING: Uncommitted changes that may be real work:"
    echo "$MEANINGFUL"
    echo "NEEDS_CONFIRMATION=uncommitted_changes"
    # Do NOT exit — let the LLM ask the user before proceeding
  fi
fi

# Check for unpushed commits (local commits not on the remote)
REMOTE_BRANCH="origin/$CURRENT_BRANCH"
if git -C "$CURRENT_WORKTREE" rev-parse --verify "$REMOTE_BRANCH" >/dev/null 2>&1; then
  UNPUSHED=$(git -C "$CURRENT_WORKTREE" log "$REMOTE_BRANCH..HEAD" --oneline 2>/dev/null)
  if [ -n "$UNPUSHED" ]; then
    echo "WARNING: Unpushed commits on $CURRENT_BRANCH:"
    echo "$UNPUSHED"
    echo "NEEDS_CONFIRMATION=unpushed_commits"
  fi
else
  # No remote tracking branch — check if there are commits beyond where main diverged
  LOCAL_ONLY=$(git -C "$CURRENT_WORKTREE" log "origin/main..HEAD" --oneline 2>/dev/null)
  REMOTE_EXISTS=$(git ls-remote --heads origin "$CURRENT_BRANCH" 2>/dev/null)
  if [ -n "$LOCAL_ONLY" ] && [ -z "$REMOTE_EXISTS" ]; then
    echo "WARNING: Branch was never pushed to remote. Local commits:"
    echo "$LOCAL_ONLY"
    echo "NEEDS_CONFIRMATION=unpushed_commits"
  fi
fi

# Switch to main if not already there
cd "$MAIN_WORKTREE"

# Graft detection: read the Graft Detection block PLUS its cleanup variant from
# ~/.claude/prompts/worktree-reference.md, then execute it here.
# Fail loudly if the file is missing — the block is required, not optional.
# The block sets USE_GRAFT, GRAFT_REPO_NAME, GRAFT_WORKTREE_NAME; the cleanup
# variant verifies graft tracks $CURRENT_WORKTREE.
#
# Read ~/.claude/prompts/worktree-reference.md now (required). If it is absent,
# set USE_GRAFT=false and warn — do NOT proceed silently with unset variables.

echo "CURRENT_WORKTREE=$CURRENT_WORKTREE"
echo "CURRENT_BRANCH=$CURRENT_BRANCH"
echo "MAIN_WORKTREE=$MAIN_WORKTREE"
echo "Now in main worktree, safe to proceed"
```

### 2. Check Merge Status (from main)

**Cache-first:** Check `.claude/github-cache.json` in the worktree before calling the GitHub API.
- If `pr.state` is `"MERGED"` → trust it (one-directional transition, always trustworthy)
- If `pr.state` is `"OPEN"` or missing → verify via API (may be stale)

```bash
# Cache-first PR state check
GITHUB_CACHE=$(cat "${CURRENT_WORKTREE}/.claude/github-cache.json" 2>/dev/null || echo '{}')
CACHED_PR_STATE=$(echo "$GITHUB_CACHE" | jq -r '.pr.state // empty' 2>/dev/null)

if [ "$CACHED_PR_STATE" = "MERGED" ]; then
  PR_STATE="MERGED"
  PR_MERGED_AT="(from cache)"
else
  # Cache says OPEN, missing, or no cache — verify via API
  PR_STATE=$(gh pr view "$CURRENT_BRANCH" --json state,mergedAt -q '.state' 2>/dev/null || echo "NONE")
  PR_MERGED_AT=$(gh pr view "$CURRENT_BRANCH" --json mergedAt -q '.mergedAt' 2>/dev/null || echo "")
fi

if [ "$PR_STATE" = "MERGED" ]; then
  echo "PR was merged (squash or regular) at $PR_MERGED_AT"
elif [ "$PR_STATE" = "CLOSED" ]; then
  echo "WARNING: PR was closed without merging"
  # Ask user for confirmation before proceeding
elif [ "$PR_STATE" = "OPEN" ]; then
  echo "WARNING: PR is still open - not merged yet"
  # Ask user for confirmation before proceeding
else
  # Fallback: No PR found, check git state
  git fetch origin --prune
  REMOTE_BRANCH="origin/$CURRENT_BRANCH"

  if ! git rev-parse --verify "$REMOTE_BRANCH" >/dev/null 2>&1; then
    echo "No PR found, but remote branch deleted - likely merged or abandoned"
  else
    echo "WARNING: No PR found and remote branch still exists"
    echo "Branch may not have been submitted yet"
    # Ask user for confirmation before proceeding
  fi
fi
```

### 2b. Check Issue Status (if branch has issue number)

**Cache-first (two caches):** Check the per-worktree `.claude/github-cache.json` first (most specific),
then the project-level `issues.json`, then fall back to the GitHub API.
- Cached `"closed"` → trust it (one-directional transition)
- Cached `"open"` or missing → verify via API

After confirming the issue is closed, update both caches.

```bash
# Detect issue from worktree cache first, then branch name
ISSUE_NUM=$(echo "$GITHUB_CACHE" | jq -r '.issue.number // empty' 2>/dev/null)
if [ -z "$ISSUE_NUM" ]; then
  # Strip any type prefix (feature/, fix/, chore/) before matching the leading number
  ISSUE_NUM=$(echo "$CURRENT_BRANCH" | sed 's|.*/||' | grep -oE '^[0-9]+' || echo "")
fi

# Check worktree cache for issue state
CACHED_ISSUE_STATE=$(echo "$GITHUB_CACHE" | jq -r '.issue.state // empty' 2>/dev/null)

if [ -n "$ISSUE_NUM" ]; then
  # Locate issues.json — layouts differ, so probe candidates in order:
  #   1. bare-repo layout:  /path/to/repo/.bare → /path/to/repo/issues.json
  #      (for a normal .git dir this resolves to the main worktree itself)
  #   2. meta-folder layout: cache sits one level above the main worktree,
  #      e.g. workspace/project/issues.json next to workspace/project/main/
  # Use --path-format=absolute: --git-common-dir alone can return a relative
  # ".git", which would silently resolve the cache to the wrong directory.
  # --path-format=absolute requires git ≥ 2.31; fall back to resolving the
  # relative path manually for older installations.
  GIT_COMMON_DIR=$(git rev-parse --path-format=absolute --git-common-dir 2>/dev/null \
    || (cd "$(git rev-parse --git-common-dir)" && pwd))
  CACHE_FILE=""
  for CANDIDATE in \
    "$(dirname "$GIT_COMMON_DIR")/issues.json" \
    "$(dirname "$MAIN_WORKTREE")/issues.json"; do
    if [ -f "$CANDIDATE" ]; then
      CACHE_FILE="$CANDIDATE"
      break
    fi
  done
  if [ -n "$CACHE_FILE" ]; then
    echo "Issue cache: $CACHE_FILE"
  else
    echo "WARN: no issues.json found in expected locations; issue-state cache will be skipped"
  fi

  # Check worktree cache first (most specific), then project-level cache
  if [ "$CACHED_ISSUE_STATE" = "closed" ]; then
    echo "Issue #$ISSUE_NUM is closed (from worktree cache)"
  fi

  CACHED_STATE=""
  if [ "$CACHED_ISSUE_STATE" != "closed" ] && [ -f "$CACHE_FILE" ]; then
    CACHED_STATE=$(jq -r ".issues[\"$ISSUE_NUM\"].state // empty" "$CACHE_FILE" 2>/dev/null)
  fi

  if [ "$CACHED_ISSUE_STATE" = "closed" ] || [ "$CACHED_STATE" = "closed" ]; then
    # Either cache says closed — trust it
    echo "Issue #$ISSUE_NUM is closed (from cache)"
  else
    # Cache says open, missing, or no cache — verify via API
    ISSUE_STATE=$(gh issue view "$ISSUE_NUM" --json state -q '.state' 2>/dev/null || echo "UNKNOWN")
    if [ "$ISSUE_STATE" = "OPEN" ]; then
      echo "WARNING: Issue #$ISSUE_NUM is still open"
      echo "PR may not have 'Closes #$ISSUE_NUM' or hasn't been merged yet"
    elif [ "$ISSUE_STATE" = "CLOSED" ]; then
      echo "Issue #$ISSUE_NUM is closed"
      # Update issues.json cache with closed state
      if [ -f "$CACHE_FILE" ]; then
        jq --arg num "$ISSUE_NUM" --arg date "$(date -u +%Y-%m-%dT%H:%M:%SZ)" \
          '.issues[$num].state = "closed" | .lastSynced = $date' \
          "$CACHE_FILE" > "${CACHE_FILE}.tmp" && mv "${CACHE_FILE}.tmp" "$CACHE_FILE"
        echo "Updated issues.json: issue #$ISSUE_NUM marked closed"
      fi
    fi
  fi
fi
```

### 2b-ii. Surface Post-Merge Manual Validations (from the issue/PR)

Merged code isn't always fully verified code — the ticket often lists checks that can only be
done by a human after deploy (visual QA, a staging flow, a metrics dashboard, an email actually
arriving). Before tearing down the worktree, read the issue and PR bodies and **raise any such
items to the user** so they aren't lost with the worktree.

Fetch the text (cheap, read-only — no confirmation needed):

```bash
# Issue title + body only (if an issue was detected in 2b). Deliberately skip comments —
# validation checklists live in the body/test plan, and comment threads can be huge
# (bots, review chatter). Cap size as a token-cost safety net.
# Use awk for a codepoint-safe byte cap (head -c can split UTF-8 sequences mid-character).
_cap8k() { awk 'BEGIN{n=0} {b=length($0)+1; if(n+b>8000){print substr($0,1,8000-n);exit} print; n+=b}'; }

if [ -n "$ISSUE_NUM" ]; then
  RAW_ISSUE=$(gh issue view "$ISSUE_NUM" --json title,body -q '.title, .body' 2>/dev/null || echo "")
  ISSUE_TEXT=$(echo "$RAW_ISSUE" | _cap8k)
  if [ "${#RAW_ISSUE}" -gt 8000 ]; then
    echo "WARN: issue #$ISSUE_NUM body truncated to ~8000 bytes for token safety"
  fi
fi

# PR body (may contain a test-plan / checklist section)
RAW_PR=$(gh pr view "$CURRENT_BRANCH" --json body -q '.body' 2>/dev/null || echo "")
PR_BODY=$(echo "$RAW_PR" | _cap8k)
if [ "${#RAW_PR}" -gt 8000 ]; then
  echo "WARN: PR body truncated to ~8000 bytes for token safety"
fi
```

Then **read the fetched text yourself** (don't grep-and-dump) and look for anything that
requires human verification after merge, e.g.:

- Sections titled "Manual validation", "Post-merge", "QA", "Test plan", "Verification",
  "Acceptance criteria", "Rollout", "Monitoring"
- Unchecked checklist items (`- [ ]`) anywhere in the issue or PR
- Phrases like "verify in staging/production", "check the dashboard", "smoke test",
  "confirm with", "after deploy"

**If any are found**, present them to the user as a short checklist and ask whether they've
been done — e.g.:

> ⚠️ The issue/PR mentions manual post-merge validations. Have you verified these?
> - [ ] Confirm the export email renders correctly in Gmail (issue #42)
> - [ ] Check the p95 latency dashboard after deploy (PR test plan)

This is **advisory, not blocking**: if the user says they're done (or wants to skip), continue.
But do wait for their answer before removing the worktree — this is the last natural checkpoint
where the context still exists. If nothing manual-validation-shaped is found, say so in one line
and move on without prompting.

### 2c. Update Project Issues Tracker

After confirming merge/close status, check if the **project root** contains an `issues.json` that is a **JSON array** with objects that have `id`, `title`, and `status` fields. If found, update the matching entry's `status` to `"done"`.

Detect `PROJECT_ROOT` per the **Project Detection** block in
`~/.claude/prompts/worktree-reference.md` (step 6 — parent of `worktrees/` structure, or the main
worktree itself).

**Matching logic** (in priority order):
1. **By id**: If `$ISSUE_NUM` matches an entry's `id` field (works for both GitHub issue numbers and local plan IDs)
2. **By branch slug**: Extract keywords from `$CURRENT_BRANCH`, match against entry titles

```bash
PROJECT_ISSUES="${PROJECT_ROOT}/issues.json"
if [ -f "$PROJECT_ISSUES" ]; then
  IS_ARRAY=$(jq 'type == "array"' "$PROJECT_ISSUES" 2>/dev/null)
  if [ "$IS_ARRAY" = "true" ] && [ -n "$ISSUE_NUM" ]; then
    # Try matching by id first
    MATCH_IDX=$(jq -r --argjson num "$ISSUE_NUM" \
      'to_entries[] | select(.value.id == $num) | .key' \
      "$PROJECT_ISSUES" 2>/dev/null | head -1)

    if [ -z "$MATCH_IDX" ]; then
      # Fallback: match by branch slug keywords against titles
      SLUG=$(echo "$CURRENT_BRANCH" | sed 's|.*/||; s/^[0-9]*-//' | tr '-' ' ')
      MATCH_IDX=$(jq -r --arg slug "$SLUG" \
        'to_entries[] | select(.value.title | ascii_downcase | contains($slug | ascii_downcase)) | .key' \
        "$PROJECT_ISSUES" 2>/dev/null | head -1)
    fi

    if [ -n "$MATCH_IDX" ]; then
      jq --argjson idx "$MATCH_IDX" '.[$idx].status = "done"' \
        "$PROJECT_ISSUES" > "${PROJECT_ISSUES}.tmp" && mv "${PROJECT_ISSUES}.tmp" "$PROJECT_ISSUES"
      MATCHED_TITLE=$(jq -r --argjson idx "$MATCH_IDX" '.[$idx].title' "$PROJECT_ISSUES")
      echo "Updated issues.json: \"$MATCHED_TITLE\" → done"
    fi
  fi
fi
```

### 2.5. Validate Merged Main (regression gate)

**Only when `PR_STATE = "MERGED"`.** For OPEN/CLOSED/NONE nothing is integrated
into `main`, so skip validation with a note. We are already in `$MAIN_WORKTREE`
(Step 1 switched there) — run everything from main, never touch the target worktree.

This catches regressions that only surface once the squash-merged PR lands
alongside other merges. **Validation never blocks cleanup**: the merge is already
irreversible, so a failure is reported loudly and we still remove the worktree.

Validation uses the **same command set `/shipit` runs** — sourced from
`.claude/repo-cache.json`, the single source of truth `/shipit` writes for this
repo's checks (`just check`, `cargo test`, `make lint`, etc. are all just entries
in it). `cleanup.md` is a global command, so it can't assume any particular tool;
the cache (or, if absent, `CLAUDE.md`) is what tells us what "green" means here.

```bash
if [ "$PR_STATE" = "MERGED" ]; then
  # 1. Update main to the merged commit (ff-only so cleanup never rewrites history)
  git pull --ff-only origin main 2>&1 \
    || echo "WARN: could not fast-forward main; validating current main state"

  # 2. Discover the check commands the SAME way /shipit does: from repo-cache.json.
  #    If it's missing we don't know what "green" means here, so we say so loudly
  #    rather than silently passing.
  REPO_CACHE=".claude/repo-cache.json"
  if [ ! -f "$REPO_CACHE" ]; then
    echo "VALIDATION=skipped — no $REPO_CACHE found."
    echo "Run /shipit once (it writes the cache) or document the check commands in"
    echo "CLAUDE.md, then validate main manually. Cleanup will continue."
  else
    # Build the command list in /shipit Step 3 order, skipping nulls. A composite
    # `check` replaces separate lint+typecheck (same rule /shipit applies).
    if [ -n "$(jq -r '.commands.check // empty' "$REPO_CACHE")" ]; then
      ORDER='["format","check","vet","test","build"]'
    else
      ORDER='["format","lint","vet","typecheck","test","build"]'
    fi
    CMDS=$(jq -r --argjson order "$ORDER" '$order[] as $k | .commands[$k] // empty' "$REPO_CACHE")

    # 3. Run each command. Warn-but-continue: a failure flags REGRESSION but the
    #    non-zero exit must NEVER abort cleanup — the merge is already irreversible.
    VALIDATION=pass
    while IFS= read -r cmd; do
      [ -z "$cmd" ] && continue
      echo "Validating merged main: $cmd"
      if ! eval "$cmd"; then
        VALIDATION=fail
        echo "FAILED: \`$cmd\` (main's integrated state after merging $CURRENT_BRANCH)"
      fi
    done <<< "$CMDS"

    if [ "$VALIDATION" = "pass" ]; then
      echo "VALIDATION=pass — merged main is green"
    else
      echo "VALIDATION=fail — REGRESSION on main; investigate separately."
      echo "Cleanup will continue (PR already merged)."
    fi
  fi
else
  echo "PR not merged (PR_STATE=$PR_STATE) — skipping main validation (nothing integrated)"
fi
```

### 3. Remove Worktree (from main)

When PR_STATE is **MERGED**, use `git branch -D` (force delete). Squash merges mean the local
branch commits never appear on main, so `git branch -d` will always fail with "not fully merged".
Since we've already confirmed the PR was merged, `-D` is safe.

When PR_STATE is anything else, use `git branch -d` (safe delete) and let it fail if not merged.

```bash
if [ "$USE_GRAFT" = "true" ]; then
  graft rm "$GRAFT_WORKTREE_NAME" --force -r "$GRAFT_REPO_NAME" 2>/dev/null \
    || echo "Worktree already removed or graft rm failed"
else
  git worktree remove "$CURRENT_WORKTREE" 2>/dev/null \
    || echo "Worktree already removed or doesn't exist"
  git worktree prune
fi

# Branch deletion — always needed (graft rm does NOT delete branches)
if [ "$PR_STATE" = "MERGED" ]; then
  git branch -D "$CURRENT_BRANCH" 2>/dev/null || echo "Branch already deleted"
else
  git branch -d "$CURRENT_BRANCH" 2>/dev/null || echo "Branch not deleted (not fully merged or already deleted)"
fi
```

## Failure Handling

| Scenario | Action |
|----------|--------|
| Only generated files dirty (`.envrc`, `.direnv/`, `.claude/`) | Auto-restore and continue |
| Real uncommitted changes | Warn and ask user before proceeding |
| No argument and not in a worktree | Abort with usage hint |
| Target resolves to main | Abort with error message |
| Target not a registered worktree | Abort with error message |
| PR confirmed MERGED | Proceed with all cleanup (including `-D`) without asking |
| Issue/PR lists manual post-merge validations | Surface as a checklist, ask user before removing worktree (advisory, never blocks if user confirms) |
| Issue/PR text unavailable (`gh` fails, no issue) | Note it and continue — don't block on a read failure |
| PR not merged (OPEN/CLOSED/NONE) | Warn, ask for confirmation before proceeding |
| PR merged — regression gate | Pull main ff-only, run `/shipit`'s `repo-cache.json` check commands |
| Validation fails after merge | Warn loudly (REGRESSION on main), continue cleanup anyway |
| Validation skipped — no `repo-cache.json` | Can't know the checks; note it and continue (run `/shipit` once to write the cache) |
| Validation skipped (PR not merged) | Nothing integrated into main — skip with a note |
| Worktree removal fails | Report error, suggest manual `git worktree remove --force` |
| **Shell stuck in deleted dir** | See recovery section below |

### Recovery: Shell Stuck in Deleted Directory

If the worktree was already deleted externally (or by a failed previous attempt), the shell's
working directory becomes invalid. ALL Bash commands will fail with "Path does not exist".

**To recover**: Ask the user to create a dummy folder at the old path, OR start a new Claude session.
Once the path exists again, the first command MUST cd to a valid permanent path (like main worktree).

## Example Usage

```bash
# From main with a glob argument:
/cleanup ../4*

# From inside the worktree (no argument needed):
/cleanup

# From the worktrees parent directory with a glob:
/cleanup ./42*

# Output:
# Resolved '../4*' → /Users/me/Repositories/my-project/worktrees/42-add-export
# CURRENT_WORKTREE=/Users/me/Repositories/my-project/worktrees/42-add-export
# CURRENT_BRANCH=feature/42-add-export
# MAIN_WORKTREE=/Users/me/Repositories/my-project/worktrees/main
# Now in main worktree, safe to proceed
# PR was merged
# Validating merged main: just check
# VALIDATION=pass — merged main is green
# ✓ Removed worktree
# ✓ Deleted branch: feature/42-add-export
```

## Notes

- This command complements `/shipit` - use `/shipit` to create PR, `/cleanup` after merge
- Before removing the worktree, the issue and PR bodies are scanned for manual post-merge
  validations (test plans, unchecked checklists, "verify in staging" notes) and raised to the
  user — the worktree is the last natural checkpoint for that context
- When the PR is merged, cleanup pulls `main` (ff-only) and runs the same checks
  `/shipit` runs — read from `.claude/repo-cache.json` — as a regression gate on
  integrated `main`. If that cache doesn't exist, validation is skipped with a note
  (run `/shipit` once to write it). Failures are reported but never block cleanup.
- Cache is committed to git, so no manual syncing needed
- Safe to run multiple times (idempotent checks)
- **Works from main or any worktree** — never cd's into the target, uses `git -C` instead
- All destructive operations run from main worktree to avoid "deleted directory" shell issues
